# 08 对讲群聊天室（聊天主页面）逻辑点梳理

仅覆盖**聊天主页面**（顶部套餐摘要、消息气泡区、底部操作区、接收规则、已读未读、群结束只读）。**仅适用于天通应急救援棒（TT_RESCUE_STICK）。**

## 一、页面结构总览
对讲群聊天主页面自上而下分三段：**① 顶部区（标题 + 状态 + 套餐摘要）→ ② 消息气泡区 → ③ 底部操作区**。
---
## 二、对讲群套餐：展示与购买
### 2.0 基本模型（地基）
- **对讲群成员都是设备**：无「人头」套餐，所有套餐口径都落到**设备**。
- **套餐归属设备池**：额度永久绑定设备、解绑不断开、快照永久同步。
- **与套餐商城解耦**：商品上下架只影响可购买性、不影响已购实例；多套餐「先到期先消耗」。
- **报位单位 = 次**：2期由「按时长」改为「按位置点次数」。
- **倒欠（负值）**：仅企业设备可倒欠。

### 2.1 入口与下钻链路
聊天页顶部套餐摘要 → 群管理·套餐概览 Tab（逐台在群设备）→ 单设备·套餐列表 → 套餐明细页；概览底部→批量购买；单设备列表底部→购买套餐（单个）。

### 2.2 四层余量口径与倒欠铁律
| 层 | 位置 | 口径 | 倒欠展示 |
|---|---|---|---|
| ① 群顶部合计 | 聊天页顶部摘要 | 全部在群设备短音 / 报位求和 | 显负数 +「欠费使用中」 |
| ② 概览逐台 | 套餐概览 Tab | 每台在群设备短音 / 报位剩余 | 显负数 +「欠费使用中」 |
| ③ 单套餐实例卡 | 套餐列表卡片 | 单台设备单个套餐实例余量 | 永不显负、只到 0 |
| ④ 套餐明细页 | 明细页 | 单实例消耗台账（逐条流水） | 纯日志 |

**铁律**：倒欠负值只在**设备聚合层（①②）**出现并标「欠费使用中」；**单套餐实例卡（③）**进度条不显倒扣、满条 100%、最低到 0，永不为负。

### 2.8 批量购买（账号维度·全部救援棒）
- **可选设备列表 = 当前对讲群内的全部在群设备**；唯一硬过滤 = 设备类型（仅救援棒）。
- **与「套餐概览」列表区别**：概览 = 仅在群设备（群维度）；批量购买 = 账号全部救援棒（账号维度）。
- **套餐选择**：本期**仅支持「同一套餐应用到所选多台」**。
- **订单模型**：**N 笔子订单 + 一次聚合支付**——每台设备独立子订单；**逐台独立激活**，失败者按一期「激活失败异步重试」，仍失败则该子订单原路退款、**不阻塞其他设备**。

### 2.9 全局规则与购买校验
- **支付无二次校验·按历史快照执行**：进入支付流程后**不做任何二次校验**，仍按下单时的**历史快照**完成支付与激活。
- **禁止跨端支付**：订单与**创建端绑定**，小程序创建的订单不能在 WEB 端支付，反之亦然。

## 四、消息气泡区（当前仅两类消息）
对讲群聊天页**当前仅两类消息**——**① 语音（短音，0x02）**、**② 报警文本（位置上报派生镜像）**。

### 4.1 两类消息与展示要素
| 消息类型 | 来源 | 经纬度 | 气泡内容 |
|---|---|---|---|
| ① 语音（短音） | 终端 PTT 上报（0x02）→ 服务端广播 | ❌ **语音不带定位** | 语音条 + 时长（1-10 秒） |
| ② SOS 报警文本 | 按键 SOS（0x01·标识 1）位置派生镜像 | ✅ **展示经纬度** | 遇到危险，触发 SOS 报警，请求支援！ |
| ② 落水报警文本 | 落水 SOS（0x01·标识 2）位置派生镜像 | ✅ **展示经纬度** | 遇到危险，触发落水报警，请求支援！ |

- **计费**：语音按短音计费（上行 1 + 下行每接收终端各 1）；报警文本镜像由位置派生，**不额外扣短音 / 报位**。

### 4.2 发送者信息（每条气泡）
- **设备图标**：每个气泡框**旁边显示发送终端的设备图标**（实时同步）。
- **昵称**：气泡**顶部显示发送终端的群内昵称**——「备注(卡号)」，无备注则展示卡号。
- **在线状态**：设备图标带在线 / 报警 / 离线角标。

### 4.3 气泡下方·已读未读实时统计
每条消息气泡下方实时展示：`X人已读　Y人未读　Z人失败`。点击展开「消息接收人列表」弹窗，按状态分三 Tab：**未读 / 已读 / 失败**。

下行状态文案（映射下行计费 §1.3 六态）：
| 下行状态 | 接收人列表文案 | 归入 Tab |
|---|---|---|
| ① 等待发送给终端 | 等待发送给终端 | 未读 |
| ② 发送至终端中 | 发送给终端中 | 未读 |
| ③ 终端未读 | 已发送，终端未读 | 未读 |
| ④ 终端已读 | 已发送，终端已读 | 已读 |
| ⑤ 下发至终端失败 | 发送失败 | 失败 |
| ⑥ 已取消 | 已取消 | 失败 |

- **统计维度 = 终端（设备）维度**：成员本质是设备终端，逐台独立统计。
- **发送方自身**：发语音的终端 A 是上行方，不计入自己的接收统计。

#### 4.3.1 失败原因透传
失败原因不另做前端枚举，以后端实现为基准：当前不预设失败原因清单，失败 Tab 按**后端返回的原因文案透传展示**。

#### 4.3.2 图标 / 昵称三处同源，但与 ACK 是两个维度
- **图标实时状态 ≠ ACK 历史状态（关键）**：图标 = 设备**此刻**全局状态（在线 / 离线 / 报警）；Tab（未读 / 已读 / 失败）= 这条消息**当时**的投递结果。二者可不一致且正常。

## 五、底部操作区与账号下发能力
### 5.1 底部按钮与权限
| 按钮 | 可见 / 权限 |
|---|---|
| 成员位置 | 所有成员可见；地图 30 秒刷新不算上报、不扣报位 |
| 群信息 | 所有成员可见，点击进群管理页「群信息」Tab |
| 添加成员（邀请） | **仅群主可见 / 可操作** |
| 购买套餐 | 任何关联账号可进入为设备购买 |

### 5.2 账号侧无消息下发入口
**已确认（产品口径）**：对讲群**账号 / 用户侧无法主动下发消息**，聊天页**不提供任何消息输入 / 发送入口**（无文本框、无语音录制按钮）。账号侧对讲群聊天页为**只读 / 监听**——仅接收并展示终端上报的语音与报警文本、做已读未读呈现，自身不发起任何下行下发。**消息来源唯一为终端上报**。

## 六、对讲群消息接收规则（账号侧）
### 6.1 账号在对讲群可接收 / 不接收
| 消息类型 | 是否进对讲群 | 依据 |
|---|---|---|
| 终端上报语音（0x02） | ✅ 终端在对讲群即进 | 路由 §四 L7~L8 |
| PTT 其他终端广播语音 | ✅ 服务端扇出至群内终端 | 路由 L8 / 下行计费 §2.3 |
| SOS 报警文本（按键 / 落水） | ✅ **镜像**进对讲群 | 路由 §5.1 L12 |
| 取消报警 / 普通位置派生文本 | ❌ 只进 SOS 求救群聊 | 路由 §5.2 L17 |
| 普通位置 / 轨迹 | ❌ 不在聊天室展示 | 接收规则 §7.1 |

### 6.2 接收对象与可见范围
- **接收对象**：对讲群成员——即**名下有设备在本群的归属账号**，以及**群主**（群主即便无自有设备在群也可见）。
- **群外关联账号不接收**：对讲群是常驻 PTT 群，非「关联即广播」。
- **企业可见范围**：企业账号只看自己创建的群。

## 七、账号侧已读 / 未读与会话项
需区分两套「已读」：**① 终端 ACK 已读**（计费 / 状态机用，0x03 / 0x05）；**② 账号侧聊天页未读**（消息列表角标 / 进入清零）。二者互不等同——账号打开聊天页 ≠ 终端已读。
| 维度 | 规则 |
|---|---|
| 会话项未读角标 | 消息列表对讲群 Tab 按最后消息时间倒序、支持未读角标 |
| 账号侧进入即读 | 账号打开对讲群聊天页 → 该账号未读清零（账号维度），**不代表终端已读** |
| 气泡级已读未读 | 每条消息气泡下方实时统计 + 接收人列表弹窗，按**终端维度**逐台展示 |
| 「全部已读」 | 只作用当前 Tab |
---
## 八、群结束 / 历史消息在聊天页（只读）
- 群结束后聊天页进入**只读**：底部管理按钮隐藏 / 置灰。
- **只读冻结口径（已确认）**：聊天页只读后，**设备图标 / 在线·报警角标与顶部设备套餐余额仍全局实时刷新**；**其余历史参数（群名、成员列表、群内昵称快照、消息记录等）定格为结束时刻快照，不再变化。**
- **普通成员归属账号**：群被群主结束时自己仍有设备在群 → 会话项`已结束`，**可进入聊天页只读查看历史**；自己设备退出 / 被移除（归零）→ 会话项灰`已退出`，**不可再进入**。
- 历史语音气泡保留并可回放。
