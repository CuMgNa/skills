# 09 星豆账户资产与运营逻辑点梳理

**核心结论**：星豆有两层身份——① 账号资产（可充值、按身份隔离、有明细与订单）；② 套餐用尽后的通信托底扣费单位。通信侧「扣谁星豆 / 欠费 / 封禁」以计费文档为准，本页只保留星豆资产侧。

## 二、星豆账户资产逻辑（账户侧·新增）
### 2.1 星豆账户归属与身份隔离
星豆首先是**账号资产**，按身份独立。**账户资产归属 ≠ 通信扣费主体**。
| 账号 / 身份 | 是否拥有独立星豆余额 | 是否可充值 | 说明 |
|---|---|---|---|
| 个人账号 | ✅ 独立 | ✅ 可充值 | 账户资产独立 |
| 企业一级账号 | ✅ 独立 | ✅ 可充值 | 账户资产独立 |
| 企业二级 / 三级子账号 | ✅ 独立 | ✅ 可充值 | 账户资产独立、不与一级共享 |
| 同手机号个人身份 + 企业身份 | ✅ 完全隔离 | ✅ 各自充值 | 余额、明细、订单均按身份隔离 |

### 2.2 星豆余额展示规则（账户侧）
| 场景 | 余额展示 | 说明 |
|---|---|---|
| 余额充足 | 展示实际余额 | 充值 / 消费后实时刷新 |
| 余额为 0 | 展示 0 | 用户侧不显示负数 |
| 子账号余额 | 展示子账号自身余额 | 不展示一级余额、不与一级共享 |

### 2.3 星豆资产全局规则
1. 星豆是系统内虚拟货币，仅限系统内使用。
2. 星豆不可提现，不支持用户主动退款。
3. 星豆余额按账号 / 身份隔离。
4. 星豆余额不显示负数（欠费表现见计费文档，不记为星豆负数）。
5. 充值星豆只增加账号余额，**不参与企业套餐欠费清偿**。
6. 星豆退还属于账户流水，不计入退款统计。
7. 后台配置变更不影响历史余额和已产生流水。

## 三、星豆充值逻辑（新增）
### 3.1 入口
| 端 | 入口 |
|---|---|
| 小程序 | 我的信息 → 星豆充值 |
| 监控平台 Web | 个人中心 / 我的信息 → 星豆充值 |
| 管理后台 | 不作为普通用户充值入口（后台仅做配置、查询、统计） |

### 3.2 充值档位
默认充值档位（按默认 1 元 = 10 豆）：50豆/5元、100豆/10元、300豆/30元、500豆/50元、1000豆/100元、自定义金额（受后台单笔最大充值金额限制，**最低充值金额为一元**）。

### 3.3 充值支付方式
- 小程序：微信小程序支付
- Web：微信扫码支付 / 支付宝扫码支付

### 3.4 充值状态机
`待支付 → 支付中 → 支付成功/支付失败/支付超时 → 星豆到账（写入星豆明细+生成成功订单）`

### 3.5 充值到账规则
| 场景 | 系统处理 | 用户侧表现 |
|---|---|---|
| 支付成功 | 增加当前账号星豆余额，生成充值流水 | 余额即时刷新 |
| 支付失败 / 取消 | 不加星豆，不生成入账流水 | 提示支付失败 / 已取消 |
| 支付轮询超时 | 订单保持待确认，后端继续以回调为准 | 90 秒后提示用户重试 |
| 重复回调 | 幂等处理，只入账一次 | 余额不重复增加 |
| 充值档位被后台禁用后 | 已创建订单按订单快照处理 | 新打开充值页不再展示该档位 |

## 四、星豆明细 / 流水逻辑（新增）
### 4.1 入口与权限
| 用户 | 入口 | 可见范围 |
|---|---|---|
| 个人账号 | 我的信息 → 星豆明细 | 仅看个人身份下的星豆流水 |
| 企业一级账号 | 我的信息 / 个人中心 → 星豆明细 | 仅看一级账号自身星豆流水 |
| 企业子账号 | 我的信息 / 个人中心 → 星豆明细 | 仅看子账号自身星豆流水；不看一级流水 |
| 管理员 | 管理后台 → 星豆明细查询 | 可查全量用户流水 |

### 4.2 用户侧星豆明细字段
交易类型（充值/消费/退还/通信扣费等）、交易额度（正数表示增加，负数表示扣减）、交易说明、交易时间、关联对象（群名称、设备卡号、订单号等）。

### 4.3 星豆流水交易类型汇总（归口映射）
| 业务动作 | 方向 | 交易类型 | 规则来源 |
|---|---|---|---|
| 星豆充值成功 | + | 充值 | 本页 §三 |
| 创建对讲群扣费 / 创建失败退还 | • / + | 消费 / 退还 | 01-创建对讲群 |
| 邀请成员扣费 / 邀请失效退还 | • / + | 消费 / 退还 | 02-对讲群添加成员邀请 |
| 上行短音 / 报位套餐不足扣星豆 | - | 通信扣费 | 06-终端上行计费 |
| 下行短音套餐不足扣星豆 / 下行失败 / 取消退还 | • / + | 通信扣费 / 退还 | 07-终端下行计费 |
| 企业欠费记账 | 无星豆流水 | — | 记设备套餐欠费，不记星豆负数 |

### 4.4 星豆流水生成原则
1. **实际扣了星豆才生成负向流水**；只扣设备套餐时不生成星豆流水。
2. **实际退了星豆才生成正向流水**；设备套餐返还不生成星豆流水。
3. 星豆返还必须原路退回**当初实际扣减的账号**。
4. 同一业务流水需具备幂等键，防止重复扣 / 重复退。
5. 退还流水与原扣费流水应有关联关系，便于对账。
6. 邀请记录模块不展示扣费 / 退还状态，仅星豆明细承载。
7. 退款统计只含套餐退款，不含星豆退还。

## 五、我的订单逻辑（星豆充值订单·新增）
### 5.1 订单类型与星豆关系
| 订单类型 | 与星豆关系 |
|---|---|
| 星豆充值订单 | 支付成功后增加账号星豆 |
| 套餐购买订单 | 只能现金支付，不扣星豆 |
| 套餐退款 | 进入退款统计 |
| 星豆退还 | 不作为退款订单，进入星豆明细 |

**退款 vs 退还 vs 欠费** 的口径：退款 = 现金套餐订单退款；星豆退还 = 创建群 / 邀请失败返还（不进退款统计）；企业欠费 = 设备套餐聚合层记负（不记星豆负数）。

## 六、后台星豆运营配置（仅星豆专属项·新增）
### 6.1 星豆专属配置项
| 配置项 | 作用范围 | 生效规则 |
|---|---|---|
| 兑换比例：1 元 = N 豆 | 星豆充值 | 修改后对新充值订单生效，不影响历史余额 |
| 充值档位 | 充值页展示 | 用户下次打开充值页时生效 |
| 单笔最大充值金额 | 自定义充值 | 提交充值订单时校验 |

### 6.2 充值档位管理
新增档位 / 禁用档位（需二次确认，禁用后灰色展示，不影响历史订单）/ 删除档位（需二次确认，已有订单关联的档位不可删除）/ 修改档位（只影响后续充值页展示和新订单）/ 历史订单（订单中冗余存储档位名称 / 星豆数 / 价格快照）。

### 6.4 订单统计口径
1. 订单销售额只统计现金订单，星豆退还不当作退款。
2. 退款统计只含套餐退款。
3. 星豆充值订单单独归类为星豆订单。
4. 企业欠费通信不计入订单销售额，按设备套餐欠费 / 负账单独统计。
---
## 七、套餐购买与星豆关系（仅边界结论·新增）
| 确认项 | 规则 |
|---|---|
| 套餐购买支付方式 | 只能现金支付（小程序微信 / Web 微信 + 支付宝） |
| 是否支持星豆支付套餐 | 不支持 |
| 是否支持现金 + 星豆混合支付 | 不支持 |
| 套餐退款 | 按现金订单退款处理，不走星豆退还 |
| 企业设备套餐欠费 | 只能通过重置 / 充值设备套餐处理，不能用星豆抵扣 |

**星豆可用于创建群、邀请成员、设备通信托底扣费，但不能购买设备套餐**。
