# 06 终端上行计费逻辑点梳理

本页梳理**天通应急救援棒（TT_RESCUE_STICK）「终端 → 平台」上行方向**的计费逻辑点。总原则：**按「上报动作」计费，不按「转发份数」计费**；转发路由不在上行侧二次扣费。

## 一、适用范围与可计费消息
- **L1**：仅适用于天通应急救援棒（TT_RESCUE_STICK）的「终端 → 平台」上行方向。
- **L2**：上行**可计费消息只有两类**——**语音（短音）**、**位置（报位）**。
- **L3**：由位置派生的报警 / 取消报警等聊天文本**不单独计费**。
---
## 二、语音上行（短音）
- **L4**：终端**上报一条语音 = 发送方设备扣 1 条短音**，仅此一次。
- **L5**：该语音转发去 SOS 求救群聊 / 对讲群 / 普通聊天室，**甚至同时去多个，都不再对发送方扣费**；发送方扣费只与「上报了一条语音」有关，与落点数量无关。
---
## 三、位置上行（报位）
核心口径：**只要是位置上报，上报一次就扣 1 次报位，与标识无关。**（v1.2 已由「按时长/分钟」改为「按位置点次数」）
| 上报类型（标识） | 上行扣费 | 逻辑点 |
|---|---|---|
| 普通位置（0） | ✅ 上报一次扣 1 次 | L6 |
| 按键 SOS（1）/ 落水 SOS（2） | ✅ 上报一次扣 1 次；同时进 SOS 求救群聊 + 对讲群**也只扣 1 次** | L7 |
| 取消 SOS（10） | ✅ 上报一次扣 1 次（同样是一次位置上报） | L8 |
---
## 四、扣费顺序（短音 / 报位**各走独立子池**·同构三步漏斗）
**套餐绑设备、不绑账号。**
| 步骤 | 扣费源 | 够扣 | 不够 → 下一步 |
|---|---|---|---|
| 1️⃣ 套餐 | 对应类型子池（语音=短音条 / 位置=报位次） | 扣 1 单位，结束 | 该子池空 → 步骤 2 |
| 2️⃣ 星豆 | 归属账号星豆（账号级单一余额） | 按单价扣（位置 1 / 语音 10），结束 | 余额 < 单价 → 步骤 3 |
| 3️⃣ 兜底 | 按 L11 锚点判定绑定类型 | — | 🏢 企业：欠费续传不中断；👤 个人：按类型部分封禁 |

- **L10**：先扣**设备绑定套餐**，按**上报类型走对应子池**。**两子池相互独立、各自计数、各自耗尽、不跨池借用**。
- **L11**：本次上报对应子池用尽后改扣**归属账号星豆**（账号级单一余额，位置 1、语音 10）。判定的**唯一锚点 = 一级账号对该设备的绑定关系是否 = 我的**（「用户」「好友」均不算企业归属）。
  - 场景1｜纯个人：一级 ≠ 我的，个人账号 = 我的 → 扣**个人账号**星豆
  - 场景2｜企业（单绑）：一级 = 我的，无个人「我的」 → 扣**一级账号**星豆
  - 场景3｜企业（双绑）：一级 = 我的，同时个人也 = 我的 → 扣**一级账号**星豆（企业优先）
  - 场景4｜无个人或企业绑定设备=我的：谁也不扣
- **L12｜星豆单价**：默认 **1 位置 = 1 星豆，1 短音 = 10 星豆**（管理后台可配）。
- **L13**：某类型**对应子池 = 0 且星豆余额 < 本次上报所需单价**时，**先按 L11 锚点判定设备绑定类型，再决定欠费或封禁**：
	- ⚖️ **总则**：封禁仅属「纯个人绑定」；**企业绑定（含双绑）永不封禁**。
	- **L13.1｜企业（一级 = 我的，含双绑）**：扣一级账号星豆；不足 → **欠费续传、上行不中断、永不封禁**。星豆余额 < 本次单价时不扣星豆，本次直接由该类型对应子池 −1 转负记欠费；一级星豆界面显示 0。
	- **L13.2｜纯个人绑定**：按类型部分封禁——某类型子池 = 0 且星豆 < 该类型单价 → **仅封该类型**，另一类型能付则继续；单类型被封时终端不播报；全停才播报：短音/条 = 0 且 报位/次 = 0 且 星豆 = 0 时播报「套餐不足请先充值，充值后重启设备」。恢复：充值/购套餐后系统自动解封（无需人工干预），但终端需重启才恢复。
	- **L13.3｜欠费回填与充值规则（仅企业）**：星豆最低为 0、永不为负；只有设备套餐可为负数。充星豆 ≠ 抵扣套餐欠费：套餐负账只能通过充设备套餐回填（如 −50 充 100 → +50）；充星豆仅作后续上行的扣费源。
---
## 六、设备归属动态变更时的计费
**核心**：套餐池（含欠费负账）按**设备**走、随归属变更由新归属方继承；归属变更**只切换三件事**——星豆扣费主体、星豆耗尽后果、设备封禁 / 欠费 / 倒欠状态。判定一律以**本次上报结算时刻的绑定快照**为准。

### 资源归属分层（什么跟设备走、什么跟账号走）
- **L16**：**套餐池（短音条 / 报位次）= 设备级**，所有关联账号共享一份；**快照一旦生成永久同步，账号解绑也不断开**。归属变更时**跟设备走、不清零、不转移**，由新归属方继承。
- **L17**：**企业历史欠费账单 = 设备级**，留在设备子池上、不随解绑清零；新归属方**不强制承担**，但任何关联账号充套餐 / 重置时**先抵扣这笔负数（= 还钱）**再计余量。
- **L18**：**星豆 = 账号级**，按当前归属实时判定扣谁。
- **L22**：**时序**——单条上报计费原子化，按**星豆结算时刻的归属快照**判定扣谁，**不回溯改判**已结算上报。
- **L23**：**返还**——套餐额度返还退回**设备池**；星豆返还的**扣费流水，原路退回当初实际扣减的账号**（即使该账号已解绑设备）。
---
## 七、边界与特殊场景
- **L14**：**无我态 / 未绑定设备**套餐耗尽后：任何上行**一视同仁**：对应子池有余则正常扣子池上行；**子池耗尽且无星豆扣费主体 → 直接停止上行、不播报、不新增任何欠费 / 倒欠**。设备级遗留账只剩「企业历史欠费」，留在设备子池、新归属不强制承担、充套餐先抵扣。
- **L15**：管理员禁用功能本期不做；本期状态只有 **不可用（欠费/封禁）> 可用** 两档。
---
## 九、一句话收口
- **计费单位**：语音上报一条扣 1 短音、位置上报一次扣 1 报位（含取消 SOS 10；SOS 报警 1/2 多路由仍只扣 1 次）。
- **扣费顺序**：先扣设备套餐对应子池（两池独立耗尽、各自落星豆），再扣归属账号星豆（企业扣一级、个人扣个人，单价可配）。
- **企业**：星豆不足走欠费、上行不停（套餐可为负，余额 < 单价时不扣星豆、套餐转负，充够单价再扣；仅充套餐清欠）。
- **纯个人**：对应子池用尽且星豆 < 该类型单价即按类型部分封禁（封禁仅属纯个人，企业含双绑永不封禁）。
- **无我态 / 未绑定**：任何上行子池耗尽且无星豆主体即停止上行、不播报、不新增账。
- **转发路由不在上行侧二次扣费。**
