# 07 终端下行计费逻辑点梳理

本页梳理 **救援棒终端「下行消息」计费逻辑点**（平台 / 服务端 → 终端方向）。**按三种下行类型分别独立说明**：普通聊天室、SOS 求救群聊、PTT 语音广播。

## 一、通用基础（三类共用，先讲一次）
### 1.1 术语
- **下行消息**：平台 / 服务端 发往救援棒终端的消息（文本 / 语音）。
- **短音**：下行计费基本单位，**1 条下行文本 / 语音入队 = 扣 1 条短音**。
- **套餐 / 星豆**：套餐额度绑「设备」，星豆绑「账号」。**设备套餐含两条相互独立的子额度：短音池（下行 / 消息用）与报位池（上行 / 位置用），各自独立计数、独立耗尽。**
- **双轨独立扣费**：短音与报位是两条平行的「子套餐 → 星豆」漏斗，互不串联。短音池 = 0 即可触发短音扣星豆（10 星豆 / 条），报位池 = 0 即可触发报位扣星豆（1 星豆 / 次）。

### 1.2 计费主体与资源闸门（按归属拆开）⭐
**下行计费先判"扣哪台设备的短音池"，再判"短音池 = 0 后有没有星豆主体兜底"。**

**催充提示是「设备级」聚合**：只有当短音与报位**两条线都不可用**时，才向终端下发统一提示「**套餐不足请先充值，充值后重启设备**」；任一条线还能用就不提示。充值后系统自动解封，但终端需重启。

归属判定表：
| 归属 | 星豆主体 | 短音池 = 0 后 | 核心边界 |
|---|---|---|---|
| 纯个人 | 个人绑定者 | 星豆 ≥ 10：扣 10 星豆可下发；星豆 < 10：短音失败 / 不入队（报位仍可用则不催充） | 个人不可欠费；短音封禁不等于报位封禁 |
| 企业（含企业双绑） | 企业一级 | 企业星豆 ≥ 10：扣 10 星豆可下发；企业星豆 0~9：不扣零散星豆，记 1 条短音欠费，仍可下发 | 企业优先；双绑也不扣个人星豆 |
| 无【我的】归属（仅用户 / 好友关系） | 无 | 无星豆兜底；短音池 = 0 后短音能力封禁 | 有关联不等于有扣费主体 |

**4）下行方向：扣哪台设备**
- 普通 / SOS → 扣**发消息那台设备**。
- PTT → 对**每台接收终端**各判一次：A 说一句，A 上行先扣 A 自己 1 条短音；广播给其他终端时，每个接收终端再各自跑一遍下行短音判断。

### 1.3 下行状态机（三类共用）
| 状态 | 触发 | 费用 |
|---|---|---|
| ① 等待发送给终端 | 扣费成功入队，设备离线 / 本轮未选中 | 已扣，暂不退 |
| ② 发送至终端中 | 通过网络把消息发给终端，等终端回『已送达』，限时 10 秒 | 已扣 |
| ③ 终端未读 | 终端已收到消息（回了『已送达』），但用户还没查看 | 已扣，不退 |
| ④ 终端已读 | 终端回『已读』（用户已查看） | 已扣，不退 |
| ⑤ 下发至终端失败 | 10 秒内没收到『已送达』回执 / 发送出错（非人为） | **退 1 条** |
| ⑥ 已取消 | 撤销仍处「等待发送」的消息（触发按类型，见第三章） | **退 1 条**（套餐过期也退） |

- 『已送达』/『已读』回执按消息编号去重（**幂等**）；重复回执不会重复扣费 / 退费。
- **取消只发生在「等待发送」窗口**；进入「发送中」后不可取消。

## 二、扣费规则（分三类独立说明）
### 2.1 普通聊天室下行
- **下行触发**：用户 / 平台在普通通信会话中**手动单发**给终端（点对点 / 会话，非群广播）。
- **谁能产生下行**：标签 = **我的 / 用户** 可发；**好友**隐藏输入框 + 后端拒绝；**无绑定历史**的会话项不可发。
- **特例·解绑后仍可下发**：个人账号与设备曾产生关联，即使之后解绑，该会话仍可继续向终端下发；但此时已无【我的】归属主体，**只扣设备套餐、不扣星豆**。
- **计费单位**：每条入队扣 **1 条短音**。
- **特点**：无「群」概念 → **无群结束类取消**，取消仅来自平台人为。

### 2.2 SOS 求救群聊下行
- **下行触发**：求救群聊**群内成员手动发**给终端。
- **计费单位**：每条入队扣 **1 条短音**。
- **特点**：群有「结束」概念（结束救援 / 报平安 / 72h 无业务消息 / 主账号解绑），直接影响退费。

### 2.3 PTT 语音广播下行（对讲群 · 服务端自动）
每个接收终端各持一份独立副本（独立扣费 / 状态机 / 退费）。
- **下行触发**：终端 A 上报 1 条 PTT 语音 → **服务端自动扇出**给同对讲群其他终端。
- **计费（两段）**：
	- **上行**：A 上报 = A 设备扣 **1 条短音，仅一次**，与广播落点数无关。
	- **下行**：广播到的**每个接收终端各扣 1 条短音**，各自按 1.2 计费主体。
	- **L5「转发去多个不再对发送方扣费」只豁免 A，不豁免接收终端。**
	- **总消耗 = 1（A 上行）+ Σ(成功入队的接收终端 × 1)**。
- **准发快照**：以下行副本生成事务时刻的群成员为本次接收终端快照；快照后新加入终端**不补收历史等待语音、不补扣费**。
- **混合成员差异**：同一对讲群可同时存在纯个人、企业、企业+个人双绑、无【我的】归属设备；按每台接收终端副本独立判断。

## 三、退费规则（分三类独立说明）
**三类共用约束**：① **只要失败就退**——不论失败原因（含发送超时），失败即退 1 条；② **原路退**：扣的是设备套餐就退回原套餐池（**套餐过期也照退**）；扣的是星豆就原路退回**原扣费归属账号**，**哪怕中途设备归属账号已动态变化**仍退给原账号；③ 同一副本只退一次·原子 + 幂等·取消只在「等待发送」窗口。

### 3.1 普通聊天室退费
- ① 下发至终端失败 → 退该终端 1 条。
- ② 已取消：**仅**应急救援平台 / 后台对「等待发送」消息**人为取消** → 原路退。

### 3.2 SOS 求救群聊退费
- ① 下发至终端失败 → 退 1 条。
- ② 已取消（对仍处「等待发送」的消息，任一触发）：求救群聊结束救援 / 报平安；报警模块处理报警；个人绑定者解绑设备；平台 / 后台人为取消。→ 退 1 条。

### 3.3 PTT 语音广播退费（逐终端独立）
- ① 下发至终端失败 → 退该终端 1 条。
- ② 已取消：仅对仍处「等待发送给终端」的接收端副本生效；进入「发送中」后即使发生离群 / 换群 / 归属变化 / 群结束，也不再取消。

**v7 口径**：群只在两种情形结束——① 群主点「结束群组」；② 群主账号注销 / 删除（企业群含上级删除导致的子级账号级联删除）。**单纯设备归零不再结束群**。

PTT 取消分类（A–F）：
- **A｜整群级取消**（群结束 → 全群「等待发送」副本逐副本原路退）
- **B｜单终端级离群**（仅取消该终端副本，群继续）：群主移除该成员设备 / 成员主动退出 / 归属账号主动解绑 / 设备类型变更为非 TT_RESCUE_STICK / 非群主账号注销 / 个人主人变更 / 群主本人设备被移除（即便归零：群继续，仅取消该设备等待副本）
- **C｜企业级联离群**（设备级，群继续）：一级群主/二级群主/三级群主的自身解绑·类型变更·换群＋父级解绑/移除分配
- **D｜换群 / 邀请并发**（旧群退出，新群不补历史）
- **E｜归属变化判定**（接收端自身·先判断是否真离群）：当前群靠哪条「我的」绑定入群，那条被删 / 换走才离群；**已扣费副本只要不离群就不重算、不改扣（锁原扣费来源）**
- **F｜平台人为取消 + 不触发取消的反例**：账号冻结 / 过期不离群（设备仍在群，仅不能发消息）；副本已进「发送中」后离群不按取消（超时即失败 → 仍退该终端 1 条）

**三类退费速查**：
| 退费触发 | 普通聊天室 | SOS 求救群聊 | PTT 语音广播 |
|---|---|---|---|
| 下发失败（非人为） | ✅ 退 1 条 | ✅ 退 1 条 | ✅ 退该终端 1 条 |
| 已取消-群结束类 | ❌ 无群概念 | ✅ 结束救援/报平安/报警处理/解绑 | ✅ 群主结束/账号删除级联 |
| 已取消-单终端离群 | ❌ | —（随群结束） | ✅ 8 条离群路径 |
| 已取消-平台人为 | ✅ 原路退 | ✅ 原路退 | ✅ 原路退 |
| 个人短音不足失败 | 本就未扣，无需退 | 同左 | 同左 |
