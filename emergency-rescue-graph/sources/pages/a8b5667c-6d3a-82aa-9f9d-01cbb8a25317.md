# SOS 群管理方案

<!-- notion_page_id: a8b5667c-6d3a-82aa-9f9d-01cbb8a25317 -->

## 1. 群管理目标与范围
- 在求救群聊中提供：
	- 群成员列表查看
	- 添加成员
	- 标记救援已完成
- 并固化成员名称/头像展示规则，支撑跨端一致的 UI 展示与测试验收。
## 2. 群成员范围（展示对象）
成员范围按 `sos-groupchat-plan_fa323615.plan.md`：
- 个人主账号 A
- 最低等级企业账号 E（设备标签为”我的/用户”的企业账号中层级最低者）
- 紧急联系人集合 C（0–3）
- 设备虚拟成员 D（用于展示设备在线/离线/报警）
- 系统客服：规则上可默认自动加入，但**本版本不考虑系统客服**（不纳入实现/测试范围）
- **版本边界说明**：对讲机ID添加成员与系统客服入群均不在本版本范围内，后续版本另行规划
补充确认：
- 群内成员**只有加入，没有退出**。
- **群成员数量无上限**。
- 冻结账号仍计入接收范围（冻结仅限制前端登录，不影响数据流转）。
## 3. 成员列表展示规则（头像 & 名称/备注）
### 3.1 设备虚拟成员（D）
- **名称来源**：默认展示**设备卡号/设备ID**（固定值）。
- **备注**：支持展示”虚拟设备备注”（如设备别名/备注名），但：
	- **设备虚拟成员名称固定不可修改**（不提供编辑入口，不接受前端改名）。
- **头像**：使用设备虚拟成员固定头像样式；并展示设备状态（在线/离线/报警）。
- **状态展示细节**：具体图标与颜色规则已在设备状态管理模块中完善，此处后续引用。
### 3.2 普通群成员（A/E/C/其他）
- **展示格式**：`名称（账号）`
	- `名称`：该成员在本群的”群内备注名/显示名”
	- `账号`：该成员的账号标识（如账号ID/手机号/对讲机ID，按端统一口径展示）
- **备注生效范围**：名称/备注**仅本群生效**（不影响其它群、不影响普通通信、不影响账号全局昵称）。
- **名称可编辑**：群管理成员行提供”编辑入口”，仅用于编辑该成员在本群的名称/备注。
### 3.2.1 名称输入约束（必测）
- **字符类型**：
	- 中文（汉字）
	- 英文字母（大小写）
	- 数字（0-9）
- **长度限制**：2–10 个字符（含边界）
- **校验行为**（建议）：
	- 不符合字符集/长度：前端阻止保存并提示；后端需二次校验并返回明确错误码/文案。
### 3.2.2 名称编辑权限（必测）
- **所有成员（含管理员）均只能编辑自己的群内名称/备注**，不存在”代改他人”权限，管理员亦无例外。
- 非本人昵称：前端不展示编辑入口；后端需做鉴权，越权请求返回明确错误码/文案。
### 3.3 自动拉群成员的默认群昵称规则
自动拉入的成员（个人主账号 A、最低等级企业账号 E、紧急联系人 C）的默认群昵称规则：
- **统一不设置昵称**：自动拉入群聊的成员默认**仅展示账号**（如账号ID、手机号等），不预填群昵称。
- **由成员自行修改**：入群后，每位成员自行在群管理中编辑**自己的**群昵称；任何人（含管理员）均无法代改他人昵称（见第 3.2.2 节编辑权限规则）。
- **展示效果**：未编辑昵称前，成员列表仅显示账号标识；编辑后按 `名称（账号）` 格式展示。
## 4. 添加成员（群管理）
### 4.1 入口与交互
- 群管理页提供「添加」按钮。
### 4.1.1 添加成员权限（必测）
- **仅个人主账号 A、最低等级企业账号 E/企业管理员**可添加成员。
- 紧急联系人及其他普通成员：无添加权限。
- 无权限时前端不展示入口（按钮隐藏或置灰不可点）；后端需做鉴权，越权请求返回明确错误码/文案。
### 4.2 添加方式（本版本）
- **仅支持手机号码添加成员**（当前不考虑对讲机ID添加）。
- 手机号码：
	- 输入手机号（必填），暂定国内手机号格式（11 位数字，1 开头）
	- 输入群昵称（必填）
	- 校验数据库是否存在对应个人账号；不存在则自动创建
	- 拉入当前求救群聊
### 4.2.2 通过手机号添加成员：账号选择与判定原则（必测）
基于 `求救群聊 — 通过手机号添加成员场景逻辑梳理.md`，固化如下原则：
- **只邀请个人账号入群**：
	- 通过手机号添加成员时，只根据手机号查找/创建**个人账号**并邀请入群；
	- 同手机号绑定的企业账号**不**通过”添加手机号”被邀请入群。
- **“已在群内”判定以个人账号ID为准**：
	- 判断某手机号对应的个人账号是否已在群内，以**个人账号ID**为准，而非手机号；
	- 企业账号的在群状态与个人账号独立判断，互不影响。
- **冻结状态不影响入群**：
	- 个人账号为冻结状态时，仍允许被邀请入群（冻结仅影响前端登录，不影响入群与消息派发）。
- **无个人账号则自动创建**：
	- 手机号无对应个人账号时，系统自动创建个人账号后邀请入群。
### 4.2.3 场景覆盖清单（建议作为测试用例直接落地）
> 说明：本版本”添加成员”输入包含：`手机号（必填） + 群昵称（必填）`。以下场景仅关注”手机号映射与入群判定”，群昵称校验见 4.2.1。
- **场景1**：企业账号A与个人账号B绑定同一手机号C，A/B均不在群内
	- 期望：通过手机号C添加 → 优先邀请个人账号B入群；不邀请企业账号A。
- **场景2**：企业账号A与个人账号B绑定同一手机号C，A在群、B不在
	- 期望：通过手机号C添加 → 仍邀请个人账号B入群（以个人账号ID判定未在群）；A在群不影响。
- **场景3**：企业账号A与个人账号B绑定同一手机号C，B已在群内
	- 期望：通过手机号C添加 → 失败；提示”用户已在群内”（以个人账号ID判定）。
- **场景4**：二级企业账号A2为最低等级企业账号已在群，个人账号B同手机号C但不在群
	- 期望：通过手机号C添加 → 邀请个人账号B入群；A2在群不影响。
- **场景5**：手机号C仅有企业账号A，无个人账号
	- 期望：通过手机号C添加 → 自动创建个人账号B并邀请入群；不邀请企业账号A。
- **场景6**：手机号C未注册任何账号（无企业/无个人）
	- 期望：通过手机号C添加 → 自动创建个人账号并邀请入群。
- **场景7**：个人账号B已以其他身份（紧急联系人/个人主账号等）被自动拉入群
	- 期望：通过手机号C添加 → 失败；提示”用户已在群内”（只要个人账号ID在群内即算）。
- **场景8**：个人账号B处于冻结状态，且当前不在群内
	- 期望：通过手机号C添加 → 仍可邀请B入群。
### 4.3 去重规则
- 若目标**个人账号ID**已在群内：**直接丢弃本次添加请求**（不重复添加、不新增系统消息）。
- **并发添加同一成员**：需加锁或幂等处理，确保多人同时添加同一手机号时，仅一次添加成功，后续请求返回”已在群内”，不重复创建账号或系统消息。
### 4.4 系统消息
- 成员加入/被补拉入群：追加系统消息记录系统时间与事件（日志性质，不参与已读未读）。
### 4.4.1 通知策略（已确认）
- 通过手机号添加成员成功后：
	- **仅在群内出现系统消息**；
	- **不做额外通知**（不发短信/站内通知/推送）。
### 4.5 添加成员后的历史消息可见性
- 新成员加入后，**可查看群内所有历史消息**（入群前的消息同步可见）。
### 4.6 添加成员后的设备关注关系（必测）
- 被添加成员（个人账号）成功入群后，系统自动将该账号与本次求救群聊对应的设备建立关注关系，关系类型为**好友**。
- 该操作由系统后台自动完成，无需被添加方主动操作。
- **幂等处理**：若该账号已关注该设备（无论当前关系类型），则跳过，不重复创建关注关系。
- 若该账号为本次自动创建的新账号，关注关系随入群操作一并建立。
- 救援完成后，关注关系**不解除**，永久保留。
## 5. 救援已完成（结束救援）
### 5.1 权限
按 `sos-groupchat-plan_fa323615.plan.md`：
- 个人主账号 A：可操作
- 最低等级企业账号 E/企业管理员：可操作
- 紧急联系人 C：不可操作（不展示按钮或置灰不可点）
- 系统客服：本版本不考虑
### 5.2 行为与结果
- 点击「救援已完成」→ 群状态变更为”救援完成”
- **个人主账号在设备管理中解绑「我的」天通救援棒** → 系统自动触发群状态变更为”救援完成”，并追加系统消息记录（例如：「xxx 解绑设备，本次救援已自动结束」）；群管理的「救援已完成」按钮同步置灰/隐藏
- 追加系统消息记录结束事件（谁/何时/何种原因）
- 全端禁发文本/语音（聊天页输入置灰/隐藏并提示）
- 取消该群会话中处于”等待发送给终端”状态的所有消息并按规则退费（详见 `sos-msg-send-limits` §4.3）
### 5.3 并发与幂等（必测）
- 多人并发触发结束：
	- 首个成功请求完成状态切换
	- 后续请求返回”已结束/幂等成功”，前端刷新状态即可
## 6. 救援完成后的群管理限制
- 群聊进入”只读”态（仅作为历史消息预览）：
	- 仍可查看成员列表与所有历史消息
	- 禁止发送消息
- **不允许再添加成员**：
	- 「添加」按钮隐藏或置灰不可点
- **不允许编辑群昵称/备注**：
	- 成员名称编辑入口隐藏或置灰不可点
- 除查看历史消息与成员列表外，群管理所有写操作均禁用
