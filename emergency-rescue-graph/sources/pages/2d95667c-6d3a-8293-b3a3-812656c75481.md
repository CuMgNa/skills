## 项目概述
### 功能概述
本 PRD 合并了星联应急叫应全端产品需求：
- **【平台WEB】**：面向个人与企业账号，提供个人中心（账号信息、实名认证、平台设置）、**我的套餐**、**套餐商城**（订购与支付）、**我的订单<br>**、**日报文下发额度**，以及**通信管理**（普通通信、求救群聊及群管理、套餐信息查看）。
- **【小程序】**：面向个人账号，提供实名认证（微信授权）、绑定设备与紧急联系人、套餐商城（我的套餐、我的订单、报文下发额度）、**通信消息<br>**（求救群聊）。
- **【管理后台】**：面向运营管理员，提供天通应急套餐管理、套餐明细查询、订单管理与退款、订单统计。
- **【APP对讲机】**：面向终端使用者，提供通讯（卫星链接、对星、发送短语、接收消息、群聊记录）、我的（平台设置 TCP、对讲机 ID 设置）。
### 范围与排除项
- **在范围内**：本 PRD 涵盖上述四端全部功能模块。
- **排除项**：控制台、多端监控、多端轨迹、设备管理、消息管理、电子围栏等模块的详细需求按端单独描述；支付渠道与清结算对接见单独文档。

### 【平台WEB】 US-001：个人中心查看与编辑
**描述**：作为平台用户，我希望在个人中心查看并编辑账号信息、资源额度与平台设置，以便管理我的账号与可用资源。
### 前置条件
- 用户已登录星联应急叫应平台。
### 触发条件
- 用户进入顶部导航下「个人中心」或从左侧个人中心菜单进入默认视图。
### 主流程
```mermaid
flowchart TD
    Start[进入个人中心] --> View[展示账号信息与资源额度]
    View --> Edit[可编辑手机号/邮箱/平台Logo/平台名称]
    Edit --> Save[保存后刷新展示]
```

**描述**：作为平台系统与群成员，我希望求救群聊的状态能随救援棒报警/报平安、超时规则及成员操作正确变更，并在「救援完成」后禁止所有人发送消息与短音，以便规范救援流程并避免无效打扰。

## 附录
### 原型图隐藏细节
1. 设备上报短音上来，但是扣除余额失败怎么办？
	- **解决方案：** 仍然在聊天界面显示，但是禁止点击播放，并在短音下方提醒充值才可以播放。充值成功扣除之前不能播放的消息数量，然后才可以播放短音。
2. 求救群聊群管理，添加成员根据原型图没发区分是对讲机还是手机号码。
	- **解决方案：** 增加一个下拉选：手机号码；对讲机。
3. 添加成员对讲机时，此时对讲机已经在别的救援中群聊中怎么处理？
	- **解决方案：** 在添加这个对讲机的群聊中添加时，提示“此对讲机已在别的求救群聊中救援，无法添加”。
4. 求救群里的系统客服怎么创建？
	- **解决方案：** 需求尚不清楚，先不做。
5. 求救群内15分钟无“有效响应”，系统是通过什么渠道通知救援机构的？
	- **解决方案：** 就这样挂着，72小时后系统自动关闭救援。
6. 恢复为普通模式，终端再次触发 SOS，是激活旧群并保留记录，还是重新创建一个新群？
	- **解决方案：** 重新创建一个新群。
7. 文本消息敏感词过滤调用第三方接口校验？
	- **解决方案：** 调用阿里云API校验。
