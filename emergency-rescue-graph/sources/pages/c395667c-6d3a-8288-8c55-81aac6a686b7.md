## 1. 引言
### 1.1 背景与目标
本期在 1 期基础上叠加 **对讲群通信、星豆虚拟货币与套餐购买、设备入库与绑定、管理后台统一运营管理** 四大产品模块。**本轮测试聚焦对讲群通信与星豆体系两大模块，**主要覆盖 **小程序（tdwt-applet）与监控平台 Web（pg-podium-tdwt）**。
测试目标：
- 验证对讲群全生命周期（创建 / 邀请 / 换群 / 消息路由 / PTT 广播 / 结束）行为正确、状态机闭环；
- 验证星豆充值、消费、套餐购买（含批量、部分失败）链路与金额准确；
- 重点覆盖 **计费幂等与防重复扣费、权限隔离、并发邀请 / 换群、SOS 与对讲群联动、套餐余额不足** 等高风险场景；
- 验证小程序与 Web 端数据同步与状态一致性。

### 2.2 不在本轮范围
> 本轮聚焦功能测试，不含独立接口测试与性能 / 压力测试。
- 独立接口测试（Postman / 接口自动化、协议级与参数级专项校验）——本轮仅通过功能操作间接验证，不做接口专项
- 性能 / 压力 / 并发容量等性能相关测试（后续专项）
- 管理员禁用、一级企业单独配置群人数上限功能本期不做
- 第三方支付通道完整验收（仅验证平台侧下单、回调、对账与失败处理）

### 3.2 测试分层与执行原则
- **P0 核心冒烟层**：充值、创建群、邀请 / 入群、PTT 语音广播、消息路由与 SOS 联动、结束群、计费扣费。
- **P1 主功能层**：群信息 / 成员管理 / 成员位置、星豆明细、多端同步。
- **P2 异常与边界层**：并发邀请、换群竞争、离线补发、余额不足、部分失败回滚、第三方失败。
- **P3 兼容与体验层**：机型 / 浏览器适配、UI 展示、地图与气泡交互。
- **核心原则**：先主链路后分支；先数据正确（账本 / DB / 日志）再看展示；每个扣费动作必验幂等；每个权限动作必验各角色视角；集成与回归全程穿插。
