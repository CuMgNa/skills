# 禅道缺陷标点符号使用规范

> 适用范围：禅道缺陷标题、缺陷正文、复现步骤、实际结果、预期结果、备注说明、接口日志描述等。
> 

> 
> 

> 核心原则：**中文描述用中文标点，技术信息用英文半角标点**；标点服务于可读、可复制、可检索。
> 

---

## 1. 中文描述使用中文标点

中文正文描述中，应使用中文标点（，。、；：“”等）。

✅ 推荐：

```
点击“确认支付”按钮后，页面无响应。
```

❌ 不推荐：

```
点击"确认支付"按钮后,页面无响应.
```

---

## 2. 技术信息使用英文半角标点

接口、JSON、SQL、日志、路径、命令、字段、枚举值等技术信息，应保留英文半角标点，避免影响复制和定位。

✅ 推荐：

```
接口 `/api/order/create` 返回 `500`。
字段 `payStatus` 返回 `null`。
订单状态由 `PAID` 变为 `WAIT_PAY`。
控制台报错：`Cannot read properties of undefined`。
```

❌ 不推荐：

```
接口“/api/order/create”返回“500”。
字段“payStatus”返回“null”。
订单状态由“PAID”变为“WAIT_PAY”。
```

---

## 3. UI 文案、按钮、页面名称使用中文引号

页面名称、按钮名称、弹窗文案、菜单名称等，使用中文引号 `“”` 标识，便于区分正文与界面元素。

✅ 推荐：

```
点击“提交订单”按钮后，页面未跳转。
进入“个人中心 > 我的订单”页面。
弹窗提示“网络异常，请稍后重试”。
```

❌ 不推荐：

```
点击提交订单按钮后，页面未跳转。
进入个人中心>我的订单页面。
弹窗提示网络异常，请稍后重试。
```

---

## 4. 接口、字段、枚举、日志使用反引号

技术对象使用反引号标识，便于研发复制和检索。

✅ 推荐：

```
接口 `POST /api/pay/confirm` 返回 `500`。
字段 `balance` 返回 `-100`。
状态值由 `ACTIVE` 变为 `EXPIRED`。
日志中出现 `NullPointerException`。
```

---

## 5. 并列词语使用顿号

多个并列模块、功能、对象之间使用顿号 `、`。

✅ 推荐：

```
该问题影响登录、支付、订单查询三个流程。
```

❌ 不推荐：

```
该问题影响登录，支付，订单查询三个流程。
```

---

## 6. 多个完整结果使用分号

当一句中包含多个完整现象时，使用分号 `；` 分隔。

✅ 推荐：

```
点击“提交”后页面无响应；刷新页面后数据未保存；重新登录后问题仍存在。
```

---

## 7. 避免连续问号、叹号、省略号

缺陷描述应保持客观，不使用情绪化标点（？？？ ！！！ 。。。 ...）。

❌ 不推荐：

```
点击按钮没反应！！！
为什么接口返回 500？？？
页面一直 loading。。。
```

✅ 推荐：

```
点击按钮后页面无响应。
接口返回 `500`。
页面持续处于 loading 状态，等待 30s 后仍未恢复。
```

---

## 8. 冒号使用规范

中文标签后使用中文冒号 `：`：

```
环境：测试环境
账号：test01
实际结果：页面无响应。
```

技术键值对、JSON、日志内容保持原始格式（英文半角）：

```json
{"code": 500, "message": "Internal Server Error"}
```

---

## 9. 路径、接口、版本号保持原样

路径、接口、版本号等不得加入中文标点改写。

✅ 推荐：

```
访问路径：`/pages/order/detail?id=123`
接口：`POST /api/order/create`
版本：`V2.3.1`
```

❌ 不推荐：

```
访问路径：/pages/order/detail？id＝123
接口：POST /api/order/create。
版本：V2。3。1
```

---

## 10. 标题不加结尾句号

缺陷标题不加句号，也不使用问号、叹号、省略号。

✅ 推荐：

```
【星豆支付】余额不足时仍可提交订单
```

❌ 不推荐：

```
【星豆支付】余额不足时仍可提交订单。
```

---

## 速查规则（可直接作为 Cursor 约束）

```
1. 中文描述用中文标点（，。、；：“”）。
2. 接口/JSON/SQL/日志/路径/命令/字段/枚举值用英文半角标点。
3. UI 文案、按钮、页面、菜单、弹窗文案用中文引号“”。
4. 接口、字段、枚举、日志、报错原文用反引号包裹。
5. 并列词用顿号、；多个完整现象用分号；。
6. 禁止连续问号、叹号、省略号（？？？ ！！！ 。。。 ...）。
7. 中文标签后用中文冒号：；JSON/日志键值保持英文半角。
8. 路径、接口、版本号保持原样，不加中文标点。
9. 标题不加结尾句号，不用问号、叹号、省略号。
```
