### 一、测试结果

本轮针对 **【天翼】星联应急叫应平台2期** 拉取「童美娜创建且未关闭」缺陷共 **31** 条。当前未关闭 20（其中回归不通过 0），已修复待回归 11，已延期 0。

- **按级别**：一级 0 / 二级 5 / 三级 21 / 四级 5
- **高优问题（二级及以上）**：5 个，建议优先修复后进入下一轮回归

**问题分类（基于未解决缺陷归纳）：**

- **【对讲群—计费与状态机类】：5 个未关闭** 涉及企业账号计费配置、批量添加未展示好友设备、已取消报警消息仍在聊天室展示、同账号重名未拦截、批量添加列表与好友设备同步缺失；典型高风险：企业账号创建对讲群仍提示消耗星豆，未与个人账号计费配置独立。
- **【群信息—生命周期与状态同步类】：3 个未关闭 + 4 个待回归** 涉及成员上限拦截、结束群后操作入口未隐藏、定时刷新缺失、邀请通知空态；其中结束对讲群后成员被自动遣散、群信息/套餐概览整页空白，直接影响历史数据可追溯性。
- **【套餐/星豆—余额与扣费类】：5 个未关闭** 涉及退款后星豆未对应扣减、自定义充值金额未联动计算、套餐明细退还记录缺失、套餐概览误减设备负数、纯个人接收终端短音扣费出现 -1 负值；存在被"白嫖"与欠费累加风险。
- **【通信/聊天—消息路由与重复转发类】：3 个未关闭 + 3 个待回归** 涉及对讲群语音未转发至绑定者普通聊天室、对讲群与普通群同时收到同一条语音、移除全部成员后群自动归类已结束、空对讲群列表预览仍显示「[语音]」。
- **【设备/账号—权限与计费封禁类】：3 个未关闭** 涉及纯个人账号无星豆无套餐时终端上下行未封禁、套餐进入欠费累加、扫码添加成员群昵称未做必填校验；纯个人账号欠费累加属高优阻断项。
- **【UI/体验—成员位置与样式类】：1 个未关闭 + 2 个待回归** 涉及底部控制面板折叠后可视区域过窄；待回归含悬浮卡样式、成员头像状态默认占位。

整体结论：未关闭缺陷集中在对讲群通信链路与星豆/套餐计费链路，**计费幂等与群生命周期状态闭环** 为本轮两大核心遗留，建议优先修复 5 个二级缺陷后再启动下一轮回归。
