## 1. 引言

### 1.1 背景与目标

本期在 1 期基础上叠加 **对讲群通信、星豆虚拟货币与套餐购买、设备入库与绑定、管理后台统一运营管理** 四大产品模块。**本轮测试聚焦对讲群通信与星豆体系两大模块，**主要覆盖 **小程序（tdwt-applet）与监控平台 Web（pg-podium-tdwt）**。

### 1.2 准入 / 准出标准

| 项 | 标准 |
| --- | --- |
| 准入 | 提测包版本号已锁定；测试用例评审通过；核心账号 / 设备 / 资金账户初始化就绪 |
| 准出 | P0 全部通过；P1 通过率 ≥ 95%；高优（二级及以上）缺陷无未关闭遗留；计费专项核对一致 |

> 本轮聚焦功能测试，不含独立接口测试与性能 / 压力测试。

- 独立接口测试（Postman / 接口自动化、协议级与参数级专项校验）

- 第三方支付通道完整验收（仅验证平台侧下单、回调、对账与失败处理）

## 3. 测试策略

### 3.1 测试类型

| 类型 | 范围 |
| --- | --- |
| 功能测试 | 对讲群、星豆、套餐、设备列表、群信息、群管理、通信消息等 |
| 兼容性测试 | 小程序 + Web 端，主流浏览器与终端机型 |

### 3.2 测试分层与执行原则

- **P0 核心冒烟层**：充值、创建群、邀请 / 入群、PTT 语音广播、消息路由与 SOS 联动、结束群、计费扣费。
- **P1 主功能层**：群信息 / 成员管理 / 成员位置、星豆明细、多端同步。
- **P2 异常与边界层**：并发邀请、换群竞争、离线补发、余额不足、部分失败回滚、第三方失败。
- **P3 兼容与体验层**：机型 / 浏览器适配、UI 展示、地图与气泡交互。

## 5. 风险评估

| 风险 | 触发条件 | 缓解措施 |
| --- | --- | --- |
| 充值退款星豆未扣减 | 第三方回调顺序异常 | 退款接口幂等校验、回调对账 |
| 短音扣费负值 | 下发失败未回滚 | 失败回调立即回滚余额 |
| 群结束操作未隐藏 | 前端状态机未闭环 | 状态机覆盖结束态分支 |
| 邀请超额 | 批量添加未拦截 | 加固上限校验 |
