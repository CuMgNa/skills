## 三、线上问题明细（均已关闭）

### L-01 对讲群收到设备上报短音无声音（分页未转 fileId）— 已修复

| 项 | 内容 |
|----|------|
| 严重级别 | P0 / 阻塞体验 |
| 模块 | 对讲群 · 消息媒体 |
| 现象 | 真实设备在对讲群上报短音后，群聊侧能收到消息，但播放无声音 |
| 根因 | 分页查询消息时未将媒体字段从 MongoDB ObjectId 转换为 **fileId** |
| 测试漏因 | **以偏概全**：SOS 有声音即放行；对讲群只核对「有无接收」，未实际点击播放听音 |
| 当前状态 | **已修复关闭** |

### L-02 终端上报过去时间导致终端状态异常 — 已修复

| 项 | 内容 |
|----|------|
| 严重级别 | P1 / 状态机 |
| 模块 | 终端定位上报 · 求救群聊生命周期 |
| 现象 | 创建并关闭求救群聊后，终端上报过去时间，导致终端状态异常 |
| 根因 | 未充分容错「采集时间 ≠ 当前时间」（过去/未来）与群聊生命周期切换的组合 |
| 测试漏因 | **异常场景不全**：忽略终端采集时间可能为过去或未来 |
| 当前状态 | **已修复关闭** |
